ArcLoom Math Loom Addendum v1.4 (Draft v0.1)
============================================
Binary‑Free Exact Arithmetic via Balanced‑Ternary Constraint Relaxation
Date: 2026-01-07

0. Scope
========

This document specifies the Math Loom (ML) subsystem for ArcLoom. ML SHALL implement exact arithmetic without binary primitives by representing numbers as balanced ternary digit strings and by evaluating operations through local constraint relaxation with deterministic settle/verify/repair rules.

1. Normative references
=======================

NF‑1: UF Spec v1.4.0 — Kernel operators L0–L4, negative‑space operator, invariants, hardening/SafeMode, and validation.
NF‑2: ArcLoom Spec v1.3 — TAL core definition, settle completion rules, tie‑break policy, and Krimelacks memory.

2. Terms and symbols
====================

T = {−1,0,+1} : balanced trit alphabet
A = (a_0, …, a_{n−1}), a_i ∈ T : operand A (least significant digit i=0)
B = (b_0, …, b_{n−1}), b_i ∈ T : operand B
S = (s_0, …, s_{n−1}), s_i ∈ T : result digits
C = (c_0, …, c_n), c_i ∈ T : carry digits (balanced)
Radix r = 3

Canonical value:
Val(A) = Σ_{i=0}^{n−1} a_i · 3^i

Local digit constraint (core of addition):
a_i + b_i + c_i = s_i + 3 c_{i+1}

3. Representation
=================

3.1 Balanced ternary digits
All Math Loom digits SHALL be encoded in the balanced set {−1,0,+1}.

3.2 Exactness
A Math Loom operation is exact if, for an n‑digit result S and carry C, the following holds:
Val(A) + Val(B) = Val(S) + 3^n · c_n
and for all i ∈ [0, n−1]:
a_i + b_i + c_i = s_i + 3 c_{i+1}.

3.3 Deterministic tie-break
If multiple (s_i, c_{i+1}) satisfy the local constraint, the implementation SHALL choose the pair that:
(1) minimizes |s_i| (prefers 0 over ±1),
(2) then minimizes |c_{i+1}|,
(3) else outputs Neutral (see §6).

4. ADD/SUB
==========

4.1 ADD
Given A, B, the goal is to find S and C satisfying the local constraints for all digits with c_0 = 0 (unless otherwise specified).

4.2 SUB
SUB(A,B) SHALL be implemented as ADD(A, Neg(B)) where Neg(B) is digit-wise negation:
Neg(B) = (−b_0, …, −b_{n−1}).

5. Execution model
==================

5.1 Constraint lattice
Math Loom SHALL map each digit position i to a local constraint element E_i coupled only to i and i+1. The physical embodiment MAY use the TAL attractor medium, but SHALL expose the same deterministic behavior as the reference model.

5.2 Bias schedule
Operands A and B SHALL be applied as biases b_A, b_B into the lattice. The schedule SHALL be bounded and reproducible given Θ_ML.

5.3 Settle completion
A gate is settled when:
• the TAL settle criterion is satisfied (ε_settle, N_settle),
AND
• all digit constraints pass verification (§5.4) within a bounded number of repair rounds.

5.4 Verification
For each digit i, verify:
a_i + b_i + c_i − s_i − 3 c_{i+1} = 0.
If any digit fails, the system SHALL enter repair.

5.5 Repair
Repair SHALL be deterministic. A conforming repair policy is:
• identify the lowest-index failing digit i (least significant first),
• locally re-bias E_i (and only neighbors) to the next tie-break option,
• re-settle locally, then re-verify.
A maximum repair round budget R_max SHALL be enforced.

5.6 Negative-space maintenance
When UF negative-space N(t)=1 holds for a declared window, Math Loom SHALL:
• re-center read thresholds for tri-stable states,
• probe tri-well separation margins,
• update drift coefficients deterministically.

6. Neutral behavior
===================

6.1 Neutral result
If Math Loom cannot settle to a verified solution within (T_settle_max, R_max), it SHALL output:
• Valid=0,
• Result digits S all 0,
• and SHALL not commit state to Krimelacks memory.

6.2 SafeMode coupling
If UF SafeMode is active, Math Loom SHALL freeze and output the neutral result.

7. Optional profiles
====================

Profile M1 (optional): MUL by convolution + normalization.
Profile M2 (optional): DIV/COMPARE via bounded iterative refinement and/or external adapter.
These profiles are optional because they depend strongly on physical implementation speed and readout bandwidth.

8. Conformance tests
====================

8.1 Exhaustive small-width correctness
For n=4 digits, test all A,B ∈ T^4 and verify:
Val(S) = Val(A)+Val(B) modulo 3^4, with carry-out c_4 captured.

PASS: 100% exact matches; no non-deterministic outcomes for same (A,B,Θ_ML).

8.2 Noise robustness
Inject bounded noise into readout thresholds and/or bias amplitude (ε levels). Verify:
• no invariant violations,
• fail cases become Neutral (Valid=0) rather than wrong results.

8.3 Drift/aging robustness
After thermal cycling / time soak, repeat 8.1. Verify negative-space maintenance restores margins and correctness.

8.4 Timing
Report settle time distribution (median/p95/p99). Fail if p99 exceeds declared budget without neutralization.
