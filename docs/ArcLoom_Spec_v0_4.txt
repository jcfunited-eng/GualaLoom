1. Introduction
---------------
ArcLoom is a deterministic architecture for <b>reality handling</b>. It does not execute instructions; instead it segments time‑stamped inputs into structural episodes (gates), injects each gate into a conservative coherence lattice, applies selectable <b>Law‑Fields</b> (constraint sets) and <b>Goals</b> (boundary terms) to shape settling, and commits stable outcomes to folded structural memory when uncertainty collapses (entropy drop). Outputs are bounded and SafeMode‑compliant.

3. System Process Flow Overview
-------------------------------
ArcLoom is one deterministic pipeline. UF L0–L4 produce gates and DSF and enforce invariants; MapInject turns gate evidence into bounded injections; ψ‑lattice evolves under laws/goals via Crank‑Nicolson; mode overlaps yield arc weights; commits update folded memory; outputs are bounded and SafeMode‑compliant.

3.1 Pipeline
------------

(0) Input Ingress + Time Manager
    raw records ─► monotonicity checks ─► jitter reorder ─► missing policy ─► sanitized stream F̃(t)

(1) UF Observability/Gating/Safety (L0–L4)
    F̃(t) ─► SEV(t)+N(t) ─► Gates Gₖ ─► ISFₖ ─► URFₖ ─► DSFₖ (bounded) + SafeMode

(2) MapInject (canonical)
    eₖ={TVRₖ,ISFₖ,URFₖ,DSFₖ,N(Gₖ),...} ─► bounded injection J(k) + integration steps N_int(k)

(3) Law‑Field + Goal selection
    choose law set 𝓛_k (min ΔS_total among relevant candidates)
    choose goal set 𝓖_k (deterministic boundary terms)

(4) ψ‑Lattice Evolution (Crank‑Nicolson)
    ψ evolves for τ=1..N_int(k) under Hermitian H_total + injection J(k)

(5) Arcs (mode overlaps)
    Aᵢ(k)=⟨Φᵢ,ψ(k)⟩  ─►  pᵢ(k)=|Aᵢ|²/Σ|Aⱼ|²

(6) Determinism + Commit
    entropy Hₖ and Detₖ ─► commit(k) if thresholds met ─► Fact Strand

(7) Krimelacks Memory
    Fact Strands ─► motif update ─► deterministic folding for locality

(8) Horizon Projection (if enabled)
    free‑evolve ψ with J=0 for H steps ─► projected arcs/DSF bundles

(9) Output Controls + Minimal Universal Adapter
    DSFₖ + arcs + memory (+ projection) ─► bounded outputs yₖ (SafeHold in SafeMode/stale)


4. Mathematics Used Page
------------------------
Normative equations and operators used in v0.4, including locked MapInject, Crank‑Nicolson integration, law selection by ΔS_total, and deterministic mode growth.

4 Operator set
--------------

4.1 UF structural operators (kernel)

∆F(t)=F̃(t)−F̃(t−Δt)
σ(t)=(1/W) Σ_{i=0}^{W−1} ||F̃(t−iΔt)−F̄_W(t)||²
κ(t)=||F̃(t+Δt)−2F̃(t)+F̃(t−Δt)||

Negative space:
N(t)=1 if σ(t)<σ_min AND ||ΔF(t)||<δ_min AND κ(t)<κ_min else 0

Boundary operator:
D(t)=α₁||ΔF(t)||+α₂σ(t)+α₃κ(t)
Boundary at t when D(t)≥τ_D

Gate:
Gₖ=[tₐ,t_b] where ∀t∈(tₐ,t_b): D(t)<τ_D

DSFₖ:
DSFₖ=(Dₖ,Mₖ,Revₖ,U*ₖ,Cₖ,Pₖ,Bₖ), Dₖ∈{−1,0,+1}

4.2 Evidence packing (normative)

eₖ = concat( norm(TVRₖ), ISFₖ, URFₖ, DSFₖ, N(Gₖ) )
All components bounded and normalized.

4.3 Injection amplitude (normative)

aₖ = clamp( a0 · wₖ · URFₖ · (1−U*ₖ) · (1−N(Gₖ)), [0, a_max] )

4.4 MapInject matrix P_in (locked OD‑4)

P_in = OrthoHashMatrix( Seed_P_in, N, dim(eₖ) )

OrthoHashMatrix procedure (deterministic):
1) Initialize counter c=0.
2) For each entry (n,j), compute u = HashToUnitFloat(Seed_P_in, n, j, c).
   Set M[n,j] = 2u − 1. Increment c.
3) Compute QR decomposition M = Q R using a fixed, specified QR algorithm.
4) Set P_in = Q (orthonormal columns).
Ties and sign conventions are fixed:
• enforce diag(R) ≥ 0 by multiplying corresponding column of Q by sign(diag(R)).

4.5 Injection field (normative)

J(k) = aₖ · normalize( P_in · eₖ )

Constraints:
||J(k)|| ≤ J_max
SafeMode ⇒ J(k)=0
N(Gₖ)=1 ⇒ aₖ ≤ a_NS_max

4.6 ψ‑lattice state

ψₙ = √ρₙ · exp(iϕₙ)

4.7 Total evolution (Hermitian)

iℏ dψ/dτ = H_total(k) ψ + J(k)

H_total(k)=H_base + H_mem + H_law(𝓛_k) + H_goal(𝓖_k) + H_safety

Hermiticity:
H_total(k)=H_total(k)†

4.8 Crank‑Nicolson integration (locked OD‑1)

For step τ→τ+1 with time step Δτ:
(I + iΔτ/(2ℏ) H_total) ψ_{τ+1} = (I − iΔτ/(2ℏ) H_total) ψ_{τ} + Δτ·J(k)

Solve deterministically with fixed solver ordering.

4.9 Mode overlaps and determinism (locked D2)

Aᵢ(k)=⟨Φᵢ,ψ(k)⟩
pᵢ(k)=|Aᵢ|² / Σⱼ |Aⱼ|²
Hₖ=−Σ_i pᵢ log pᵢ
Detₖ=1 − Hₖ/H₀

Commit:
commit(k)=1 iff Detₖ≥Det_commit AND max_i pᵢ(k)≥p_commit

4.10 Mode bank growth (locked OD‑5)

When commit(k)=1:
Snap(k) = {ψ(k−s)...ψ(k)} for s in [0..S_cap]
Φ_new = normalize( Σ_{ψ in Snap(k)} ψ )
Insert iff novelty(Φ_new, Φ_i) ≥ τ_novel for all i.
Tie‑break: lowest index.

4.11 Law selection by minimum ΔS_total (locked OD‑6)

Define candidate law sets ℒ_cand filtered by relevance:
relevant(ℒ, k)=1 iff score_relevance(ℒ, DSFₖ, memory, hints) ≥ τ_rel.

Define scalar total action surrogate:
S_total(ℒ) = ⟨ψ| (H_base+H_mem+H_law(ℒ)+H_goal+H_safety) |ψ⟩

Select:
𝓛_k = argmin_{ℒ in ℒ_cand AND relevant(ℒ,k)=1}  [ S_total(ℒ) − S_total(ℒ_null) ]

If no relevant laws exist ⇒ 𝓛_k = ℒ_null.

4.12 Horizon projection (optional when enabled)

Free evolve H steps with J=0 and fixed (𝓛_k,𝓖_k):
ψ̂(k+h)=FreeEvolve(ψ̂(k+h−1))
Compute p̂ᵢ(k+h) and Det̂(k+h).

4.13 Output bounds

||yₖ|| ≤ Y_max
SafeMode or stale ⇒ yₖ = Y_hold = 0 vector (neutral hold)


5. Time Management
------------------
Manages three clocks (t,k,τ) plus horizon steps h with deterministic scheduling and bounded latency using drift bundles and TTL.

5 Time rules
------------

5.1 Lattice step allocation per gate (normative)

N_int(k) = clamp_int( N_min + round( N_range · Sched(k) ), [N_min, N_max] )

Sched(k) = (1 − U*ₖ) · wₖ · (1 − N(Gₖ))

SafeMode ⇒ N_int(k)=N_min

5.2 Drift bundles (normative)

Maintain gate baselines μ over last W_g gates:
DriftVec(k) = (μₖ−μₖ−1, μₖ−μₖ−2, ...)

5.3 Output staleness (normative)

stale(yₖ)=1 iff (k−k_emit) > TTL_g
stale ⇒ yₖ = Y_hold


6. Input Integrity
------------------
Deterministic pre‑error checking and negative‑space checks prevent corrupt inputs from destabilizing UF, laws, lattice, memory, or outputs.

6 Checks
--------

6.1 Health checks (normative)

Invalid record if ANY:
• t_i ≤ t_{i−1}
• ||ΔF(t)|| > Δ_max
• σ(t) ∉ [0, σ_max)
• F(t) ∉ D ⊂ ℝᵈ

6.2 Timestamp anomaly policy (locked OD‑2)

Reorder within jitter window J_max:
• Maintain a buffer of records with t in [t_now − J_max, t_now].
• Output records in strictly increasing t order.
If violations exceed V_max within window W_v:
• trigger SafeMode.

6.3 Negative‑space (normative)

N(t)=1 if σ(t)<σ_min AND ||ΔF(t)||<δ_min AND κ(t)<κ_min else 0
N(Gₖ)=1 if mean_{t∈Gₖ} N(t) ≥ ν_NS else 0

If N(Gₖ)=1 then:
• aₖ ≤ a_NS_max
• optional: reduce N_int(k) by factor f_NS

11 Goals
--------
Goals and boundary terms bias ArcLoom toward desired readouts while remaining bounded and SafeMode‑compliant. In SafeMode, goals are disabled.

11 Goal block
-------------

11.1 Goal representation (normative)

Goal set:

11.2 Canonical goal with readout operator

Choose bounded O_out:
y_pred = O_out ψ

11.3 Safety

SafeMode ⇒ H_goal = 0
||H_goal|| ≤ H_goal_max

14 Memory
---------
Krimelacks memory stores committed strands as motifs and deterministically folds locality so structurally related motifs remain near each other.

14 Memory block
---------------

14.1 Updates (normative)

W_adj(u,v) ← W_adj(u,v) + η_mem · score(u,v,strand)
W_adj(u,v) ← (1−λ_mem) · W_adj(u,v)

Deterministic merge:
if W_adj(u,v) ≥ τ_merge then merge(u,v) (tie‑break: lowest id)

14.2 Folding (normative)

Cost(pos)=Σ_{u,v} W_adj(u,v)·dist(pos(u),pos(v))
Update pos to reduce Cost within ε_fold under deterministic update order.
